## Tutorials Definition
Takes the reader through a series of steps to complete a particular task. The task is something that you decide. Once the reader has finished a tutorial, they can better understand the rest of the documentation and have a better understanding of the overall project. By finishing a tutorial successfully, they can feel a sense of accomplishment.

Tutorials should:
* be concrete, not abstract—they're for the specific, not the general
* only explain what absolutely matters, explanation doesn't matter when someone is learning a new skill
* no distractions, don’t give options or choices, you decide what the user will do and what the outcomes are going to be

## Notes
* Should be for people new to meshtastic trying to get up to speed.  
* Should focus on helping the reader learn using specific and practical steps.  

## Examples
* How to set a node up for the first time
* How to use HeyWhatsThat to make a splash map
